Workflows

Workflows allow you to chain rules together to form sequences of actions.

While a single rule can fill forms on a single page, workflows continue to execute even after a page reload.

Use the "Workflows" menu on the left side to create workflows and add existing rules in an arbitary order.
You can drag and drop those workflow steps to reorder them.

Notice that a workflow by itself doesn't submit any forms or changes or reloads the page url.
This is something you must do at the end of every rule.

Usually you can archieve this by using a value function that clicks on a link or button.

Example rule that clicks on a submit button: 

  {
    name: "a clicking rule",
    url: /some-url/,
    fields: [{
      selector: "form input[type=submit]",
      value: Libs.h.click
    }]
  }

If you run a workflow and strange things happen (like you can't execute rules after that), try the button in the Workflows panel.

Executing workflows

Once you have defined a workflow it will be shown in the rules popup if the condition of the first rule in the workflow matches.
Click on the workflow will execute it
